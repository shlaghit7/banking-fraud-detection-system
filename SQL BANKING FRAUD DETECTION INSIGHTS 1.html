🧱 2. Database Schema (Professional Design)

🔹 Customers Table

SQL
CREATE TABLE customers (
    customer_id INT PRIMARY KEY,
    name VARCHAR(100),
    dob DATE,
    city VARCHAR(50)
);

🔹 Accounts Table

SQL
CREATE TABLE accounts (
    account_id INT PRIMARY KEY,
    customer_id INT,
    account_type VARCHAR(20),
    balance DECIMAL(12,2),
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);

🔹 Branches Table

SQL
CREATE TABLE branches (
    branch_id INT PRIMARY KEY,
    branch_name VARCHAR(100),
    city VARCHAR(50)
);

🔹 Transactions Table

SQL
CREATE TABLE transactions (
    transaction_id INT PRIMARY KEY,
    account_id INT,
    amount DECIMAL(10,2),
    transaction_type VARCHAR(20),
    location VARCHAR(50),
    transaction_date DATETIME,
    FOREIGN KEY (account_id) REFERENCES accounts(account_id)
);

🔹 Fraud Alerts Table

SQL
CREATE TABLE fraud_alerts (
    alert_id INT PRIMARY KEY AUTO_INCREMENT,
    transaction_id INT,
    alert_type VARCHAR(50),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

📈 4. Core SQL Queries

🔹 All Transactions

SQL
SELECT * FROM transactions;

🔹 Customer-wise Summary

SQL
SELECT c.name, SUM(t.amount) AS total_transaction
FROM customers c
JOIN accounts a ON c.customer_id = a.customer_id
JOIN transactions t ON a.account_id = t.account_id
GROUP BY c.name;

🔹 Account Balance

SQL
SELECT account_id,
SUM(CASE 
    WHEN transaction_type='Deposit' THEN amount
    ELSE -amount END) AS balance
FROM transactions
GROUP BY account_id;

🚨 5. Fraud Detection Queries

🔥 High Value Transactions

SQL
SELECT * FROM transactions
WHERE amount > 100000;

👉 Detects unusually large transactions

⚡ Multiple Transactions in Short Time

SQL
SELECT account_id, COUNT(*) 
FROM transactions
WHERE transaction_date >= NOW() - INTERVAL 10 MINUTE
GROUP BY account_id
HAVING COUNT(*) > 3;

👉 Detects rapid activity

🌍 Different Locations Quickly

SQL
SELECT account_id
FROM transactions
GROUP BY account_id
HAVING COUNT(DISTINCT location) > 2;

👉 Detects suspicious location jumps

📊 6. Insights Queries

🏆 Top Customers

SQL
SELECT c.name, SUM(t.amount) AS total
FROM customers c
JOIN accounts a ON c.customer_id=a.customer_id
JOIN transactions t ON a.account_id=t.account_id
GROUP BY c.name
ORDER BY total DESC
LIMIT 5;

🔥 Most Active Accounts

SQL
SELECT account_id, COUNT(*) AS txn_count
FROM transactions
GROUP BY account_id
ORDER BY txn_count DESC;

🧪 Model: Isolation Forest (Best for Fraud)

Python
import pandas as pd
from sklearn.ensemble import IsolationForest

# Load dataset
df = pd.read_csv("transactions.csv")

# Features
X = df[['amount']]

# Model
model = IsolationForest(contamination=0.05)

df['fraud_flag'] = model.fit_predict(X)

# Convert output
df['fraud_flag'] = df['fraud_flag'].apply(lambda x: 1 if x == -1 else 0)

print(df.head())

🧠 What This Does

  • Learns normal transaction patterns

  • Flags outliers as fraud
